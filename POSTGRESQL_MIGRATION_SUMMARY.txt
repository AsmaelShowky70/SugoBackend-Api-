🚀 خطوات النشر النهائية على Railway (موجزة)
═══════════════════════════════════════════════════════════════════════════════

1. ادفع الكود:
   git add .
   git commit -m "PostgreSQL integration + Railway optimizations"
   git push origin main

2. في Railway Dashboard:
   - اذهب إلى Project > Variables
   - أضف هذه المتغيرات:

   ┌─────────────────────────────────────────────────────────────────────┐
   │ ASPNETCORE_ENVIRONMENT                  = Production                │
   │ Jwt__Key                                = [مفتاح قوي >= 32 حرف]     │
   │ ConnectionStrings__DefaultConnection    = [الرابط من Railway DB]    │
   └─────────────────────────────────────────────────────────────────────┘

3. إذا لم تكن لديك PostgreSQL على Railway:
   - Dashboard > Plugins
   - أضف: PostgreSQL (سيتم إنشاء DB تلقائياً)
   - انسخ CONNECTION_URL إلى ConnectionStrings__DefaultConnection

4. الضغط على Deploy (إعادة النشر)
   - Railway ستبني المشروع من جديد
   - راقب Logs (اتبع RAILWAY_DEPLOYMENT_AR.txt للتفاصيل)

5. اختبر:
   - افتح https://your-app.railway.app/health
   - يجب أن تحصل على: {"status":"Healthy",...}

⚠️ نقاط حرجة (يجب الانتباه)
═══════════════════════════════════════════════════════════════════════════════

1. Jwt__Key:
   ✗ لا تستخدم القيمة الافتراضية
   ✓ استخدم مفتاح عشوائي قوي >= 32 حرف (استخدم password generator)

2. ConnectionStrings__DefaultConnection:
   ✗ لا تترك sqlite connection string
   ✓ استخدم الرابط الكامل من Railway PostgreSQL

3. ASPNETCORE_ENVIRONMENT:
   ✗ لا تستخدم "Development" في Production
   ✓ استخدم "Production" دائماً

4. PORT:
   ✗ لا تضع PORT يدوياً في متغيرات Railway
   ✓ اترك Railway تضبطه تلقائياً (في docker-compose يستخدم ${PORT:-5000})

5. CORS:
   ✗ لا تترك AllowAll في Production
   ✓ حدّد نطاقات العميل فقط (عدّل CORS policy في Program.cs)
